TRPG / Murder Mystery Translator 사용 설명서

일본어 TRPG 시나리오 및 머더 미스터리 텍스트를 한국어로 번역·교정하고,
코코포리아 룸 이미지를 번역본으로 교체하는 도구입니다.

[그림 1] 메인 화면 — 5개 탭 구성

1. 설정 (Settings)

  1. API Key: Google Gemini API Key를 입력하고 저장하세요. (최초 1회)
  2. 모델 이름: 기본값은 gemini-3-flash-preview입니다. 다른 모델은 Google AI 공식 문서에서 확인하세요.
  3. 버전 정보: 버전 정보 버튼으로 최신 패치 노트를 확인할 수 있습니다.

2. 1차 번역 (Translation) 핵심 기능

PDF, TXT, 이미지 파일을 일괄 번역합니다. 용어집을 활용하면 고유명사를 일관성 있게 번역할 수 있습니다.

[그림 2] 1차 번역 탭 — 전체 설정 화면

입력 설정

  • 텍스트 폴더: .pdf, .txt 파일이 있는 폴더를 선택합니다.
  • 이미지 폴더: .png, .jpg 이미지 폴더를 선택합니다. (OCR 번역)
  • 출력 폴더: 번역 결과물이 저장될 폴더입니다. 하위 폴더 구조가 그대로 유지됩니다.

고유명사 자동 추출 New v0.4

  • 번역 시작 전 용어집 추출 버튼을 누르면 AI가 시나리오를 스캔해 인명·지명·고유용어를 원문:역문 형식으로 자동 추출합니다.
  • 추출된 파일은 auto_glossary.txt로 저장되며 용어집 경로에 자동 설정됩니다.
  • 이후 번역 시 해당 용어가 우선 반영되어 일관성이 향상됩니다.

[그림 3] 고유명사 자동 추출 결과 예시 (auto_glossary.txt)

PDF 이미지 내보내기 v0.3

PDF에 포함된 이미지를 원본 해상도 그대로 파일로 추출합니다. 이미지 글자 제거나 코코포리아 교체 작업 시 필요한 원본 이미지를 손쉽게 확보할 수 있습니다.

[그림 4] PDF 이미지 내보내기 결과 — 원본 해상도로 저장된 이미지들

PDF 번역 시 이미지 선택 v0.3

PDF 파일을 번역할 때 이미지 선택 팝업이 표시됩니다. 번역이 필요한 이미지만 체크하여 불필요한 API 비용과 시간을 절약할 수 있습니다.

[그림 5] PDF 이미지 선택 화면 — 필요한 이미지만 선택하여 번역

DOCX 출력 New v0.4

  • 번역 결과를 워드 문서(.docx)로 저장합니다.
  • 원본의 글자 색상, 볼드(굵기), 하이라이트(형광펜) 서식을 보존하여 출력합니다.
  • 폰트 설정에서 원하는 폰트를 지정할 수 있습니다. (기본값: 바탕)
※ 주의: 원본의 글자 색상·볼드·하이라이트를 보존하려면 반드시 DOCX 출력을 활성화해야 합니다. HTML 결과물에는 색상 정보가 포함되지 않습니다.

3. 2차 교정 (Correction)

1차 번역된 결과물을 AI가 재검토하여 맞춤법·문맥·자연스러움을 다듬습니다.

[그림 7] 2차 교정 탭 화면

4. 이미지 글자 제거 (beta) New v0.5

AI를 이용해 이미지 배경을 보존하면서 텍스트만 제거합니다. 타이틀 로고 제거나 배경 이미지 정리에 유용합니다.

⚠️ 비용 주의: 이 기능은 이미지 픽셀 수에 비례하여 API 비용이 청구됩니다. 고해상도 이미지나 대량 처리 시 비용이 크게 증가할 수 있으므로, 꼭 필요한 이미지에만 사용하세요. 처리 전 반드시 한 장 미리보기로 결과를 확인하는 것을 권장합니다.
  • 지시 사항: "캐릭터 얼굴은 건드리지 마" 등 구체적인 추가 지시가 가능합니다.
  • 한 장 미리보기: 처리 전 결과를 비교하여 확인합니다.
  • 알파 채널 유지: 체크 시 투명 배경(PNG)의 투명도를 보존합니다.

5. 코코포리아 룸 셋팅 New v0.7

코코포리아(ccfolia) 룸 ZIP을 압축 해제한 폴더 안의 이미지를 번역본으로 교체하고, __data.json의 SHA-256 해시 참조를 자동으로 업데이트합니다.

[그림 10] 코코포리아 룸 셋팅 탭 전체 화면

경로 설정

항목설명
원본 폴더코코포리아 룸 ZIP 압축 해제 폴더 (__data.json + 해시명 이미지 포함)
번역본 폴더번역된 이미지 파일들이 있는 폴더
출력 폴더교체 완료된 결과물이 저장될 폴더

매칭 방식 선택

① 수동 매핑

번역본 이미지마다 대응하는 원본 이미지를 직접 시각적으로 선택하는 방식입니다.

[그림 11] 수동 매핑 + API 자동 매칭 옵션

  • API 자동 매칭: 체크 시 Gemini Vision이 번역본과 가장 유사한 원본 이미지를 미리 추천해줍니다. 수동으로 수정도 가능합니다.
    • 판단 기준: ① 텍스트 내용 일치(한국어↔일본어) → ② 색상·구도 유사도
    • 모델 직접 입력 가능 (기본값: gemini-3.1-flash-lite-preview)

[그림 12] 수동 이미지 매핑 화면 — 번역본(좌) → 원본(우) 매핑

[그림 13] 원본 이미지 선택창 — 클릭하여 선택

Tip: 매핑 결과를 JSON 파일로 저장/불러오기할 수 있어, 다음 작업 시 재사용이 가능합니다. 원본이 지정되지 않은 이미지는 자동으로 건너뜁니다.

② 파일명 자동 매칭

원본 폴더와 번역본 폴더에서 파일명이 동일한 것끼리 자동으로 연결합니다. 이미지 글자 제거 기능을 사용했을 때 파일명이 유지되므로 이 방식으로 바로 매칭할 수 있습니다.

[그림 14] 파일명 자동 매칭 선택 화면

[그림 15] 자동 매칭 결과 확인 — 체크박스로 개별 선택/해제 가능

출력 옵션

  • ZIP으로 출력: 체크 시 출력 폴더를 .zip으로 압축합니다. 코코포리아 업로드 시 바로 사용할 수 있습니다.

[그림 16] 이미지 교체 완료 후 로그 출력

※ 동작 원리: 코코포리아 이미지 파일명은 파일 내용의 SHA-256 해시값입니다. 번역본 이미지의 새 해시를 계산해 __data.json 전체에서 구 해시를 일괄 치환합니다. 배경·카드·마커 등 모든 참조가 자동으로 업데이트됩니다.
Tip: 결과물(output 폴더)은 웹브라우저나 워드 프로세서로 열어 확인할 수 있습니다.
